# Dark Mode Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a warm dark mode to blossomstudios.dev with a header toggle, OS-preference default, FOUC-free first paint, and cross-tab sync — all without touching component-level code.

**Architecture:** Override Tailwind v4 `@theme` CSS variables inside `[data-theme="dark"]` blocks in `src/styles/global.css`. An inline script in `<head>` sets `data-theme` on `<html>` synchronously before paint, reading from `localStorage.theme` first and falling back to `prefers-color-scheme`. A new `ThemeToggle` component in the header writes to `localStorage`, updates the attribute, and listens for `storage` events to keep tabs in sync.

**Tech Stack:** Astro 6, Tailwind CSS v4 (`@tailwindcss/vite`), CSS custom properties, vanilla JS in `<script is:inline>` blocks.

**Design Doc:** [docs/plans/2026-05-03-dark-mode-design.md](2026-05-03-dark-mode-design.md)

## Task 4: Inline FOUC-prevention script in Head.astro

**Files:**
- Modify: `src/components/Head.astro` — insert before line 29 (`<meta charset="utf-8" />`)

**Why:** Without this, on first paint the page renders in the default (light) theme, then JavaScript runs and flips it. The visible flash is jarring. The script must be inline (not module/external) so it executes synchronously before stylesheets apply.

**Step 1: Add the inline script**

Insert at the very top of the markup section (right after the closing `---` of the frontmatter, before `<meta charset>`):

```astro
<script is:inline>
  (function () {
    try {
      var stored = localStorage.getItem('theme');
      var theme = stored || (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
      document.documentElement.setAttribute('data-theme', theme);

      // If user hasn't explicitly chosen, follow OS changes live.
      if (!stored && window.matchMedia) {
        window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', function (e) {
          if (!localStorage.getItem('theme')) {
            document.documentElement.setAttribute('data-theme', e.matches ? 'dark' : 'light');
          }
        });
      }
    } catch (e) {
      // localStorage may throw in private mode; fall back to light.
      document.documentElement.setAttribute('data-theme', 'light');
    }
  })();
</script>
```

**Step 2: Verify FOUC is gone**

Hard reload (Cmd+Shift+R). Expected: no flash — page paints directly in the correct theme.

In DevTools → Application → Local Storage → `http://localhost:4321`: clear the `theme` key. Then in DevTools → Rendering → set `prefers-color-scheme` to `dark`. Hard reload. Expected: page comes up in dark mode immediately.

Now toggle the rendering preference to `light`. The page should flip live (because no `localStorage.theme` is set).

**Step 3: Commit**

```bash
git add src/components/Head.astro
git commit -m "Inject FOUC-prevention theme script into Head"
```

## Task 5: Create the ThemeToggle component

**Files:**
- Create: `src/components/ThemeToggle.astro`

**Step 1: Write the component**

```astro
---
// Theme toggle button — flips between light and dark, persisting to localStorage.
// The actual theme value is set by the inline script in Head.astro before paint.
---

<button
  type="button"
  id="theme-toggle"
  class="inline-flex items-center justify-center w-9 h-9 rounded-lg text-ink-muted hover:text-ink hover:bg-surface-raised transition-colors duration-200"
  aria-label="Switch theme"
  title="Switch theme"
>
  <!-- Moon icon (shown in light mode) -->
  <svg
    class="theme-icon-moon w-4 h-4"
    viewBox="0 0 16 16"
    fill="none"
    stroke="currentColor"
    stroke-width="1.5"
    stroke-linecap="round"
    stroke-linejoin="round"
  >
    <path d="M13.5 9.5A5.5 5.5 0 0 1 6.5 2.5a.5.5 0 0 0-.7-.5A6.5 6.5 0 1 0 14 10.2a.5.5 0 0 0-.5-.7Z" />
  </svg>
  <!-- Sun icon (shown in dark mode) -->
  <svg
    class="theme-icon-sun w-4 h-4 hidden"
    viewBox="0 0 16 16"
    fill="none"
    stroke="currentColor"
    stroke-width="1.5"
    stroke-linecap="round"
    stroke-linejoin="round"
  >
    <circle cx="8" cy="8" r="3" />
    <path d="M8 1v2M8 13v2M1 8h2M13 8h2M3.5 3.5l1.4 1.4M11.1 11.1l1.4 1.4M3.5 12.5l1.4-1.4M11.1 4.9l1.4-1.4" />
  </svg>
</button>

<style>
  /* Reveal the right icon based on current theme */
  :root[data-theme="dark"] .theme-icon-moon { display: none; }
  :root[data-theme="dark"] .theme-icon-sun { display: inline-block; }
  :root:not([data-theme="dark"]) .theme-icon-sun { display: none; }
  :root:not([data-theme="dark"]) .theme-icon-moon { display: inline-block; }
</style>

<script is:inline>
  (function () {
    var btn = document.getElementById('theme-toggle');
    if (!btn) return;

    function currentTheme() {
      return document.documentElement.getAttribute('data-theme') === 'dark' ? 'dark' : 'light';
    }

    function setTheme(next) {
      document.documentElement.setAttribute('data-theme', next);
      try { localStorage.setItem('theme', next); } catch (e) {}
      updateAria(next);
    }

    function updateAria(theme) {
      var label = theme === 'dark' ? 'Switch to light mode' : 'Switch to dark mode';
      btn.setAttribute('aria-label', label);
      btn.setAttribute('title', label);
    }

    updateAria(currentTheme());

    btn.addEventListener('click', function () {
      setTheme(currentTheme() === 'dark' ? 'light' : 'dark');
    });

    // Cross-tab sync: another tab changed the theme, mirror it here.
    window.addEventListener('storage', function (e) {
      if (e.key === 'theme' && (e.newValue === 'light' || e.newValue === 'dark')) {
        document.documentElement.setAttribute('data-theme', e.newValue);
        updateAria(e.newValue);
      }
    });
  })();
</script>
```
